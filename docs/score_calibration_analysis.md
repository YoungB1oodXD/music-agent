# 推荐分数校准分析

## 一、问题背景

当前推荐系统返回的原始分数（similarity / hybrid_score / cf_score）直接乘以 100 后显示为"匹配指数"，导致大多数推荐卡片显示 20%~30%，观感很差。

## 二、原始分数分析

### 2.1 Semantic Similarity 分布

| 指标 | 值 |
|------|-----|
| 理论范围 | 0.0 ~ 1.0 |
| 推荐阈值 (`_MIN_SEMANTIC_SIMILARITY`) | 0.26 |
| 最低硬阈值 (`_HARD_MIN_SEMANTIC_SIMILARITY`) | 0.18 |
| 实际有效范围 | 0.18 ~ 1.0 |
| 典型高分区间 | 0.35 ~ 0.55 |
| 典型低分区间 | 0.18 ~ 0.30 |

**关键发现**：
- 通过阈值的最低 similarity = 0.26，乘以 100 = 26%
- 典型高分 0.35 ~ 0.55，乘以 100 = 35% ~ 55%
- 这就是为什么大多数卡片显示 20%~50% 的原因

### 2.2 Hybrid Score 分布

Hybrid score 经过 min-max 归一化：
```
normalized = (raw - min_score) / (max_score - min_score)
```

**特点**：
- 范围是 0.0 ~ 1.0
- 是批内相对分数，不是绝对分数
- 如果候选集中分数差距大，归一化后的分布会更分散

### 2.3 CF Score 分布

协同过滤分数取决于用户行为数据，通常在 0 ~ 1 范围内。

## 三、为什么原始分数不适合直接展示

1. **语义不匹配**：
   - similarity 表示"向量相似度"，不是"推荐匹配度"
   - 用户期望"95% 匹配"表示"很适合你"，而不是"向量夹角很小"

2. **数值分布问题**：
   - similarity 有效范围 0.18~1.0，但典型推荐在 0.26~0.55
   - 直接乘 100 导致大量 20%~50% 的显示

3. **用户心理预期**：
   - 用户期望推荐结果应该在 75%~95% 区间
   - 低于 60% 会显得系统"不自信"或"不好用"

4. **排序 vs 展示**：
   - 原始分数用于排序，只需要相对大小正确
   - 展示分数需要符合用户直觉和产品语义

## 四、校准策略选择

### 方案对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| 简单线性映射 | 简单 | 分数分布可能不均 |
| Min-Max 批内映射 | 批内分层好 | 跨批不稳定 |
| 分段映射 | 可控 | 需要调参 |
| **混合策略** | 平衡 | 复杂度适中 |

### 选择：混合策略

1. **基础分数**：基于排名的位置分数
2. **原始分调整**：考虑原始分数的相对差异
3. **范围约束**：确保在 65~95 区间

## 五、映射规则

### 5.1 原始分数语义

| raw_score 范围 | 实际含义 | 不适合展示原因 |
|----------------|----------|----------------|
| 0.18 ~ 0.26 | 勉强通过阈值 | 显示为 18%~26%，太低 |
| 0.26 ~ 0.35 | 一般匹配 | 显示为 26%~35%，观感差 |
| 0.35 ~ 0.50 | 较好匹配 | 显示为 35%~50%，仍偏低 |
| 0.50 ~ 0.70 | 良好匹配 | 显示为 50%~70，可接受 |
| 0.70 ~ 1.0 | 优秀匹配 | 显示为 70%~100，合理 |

### 5.2 display_score 校准函数

```python
def calibrate_display_score(raw_score: float, rank: int, total: int) -> int:
    """
    校准展示分数，使其符合用户直觉。
    
    规则：
    1. Top 1 位置分: 95
    2. 最后位置分: 75
    3. 中间按排名线性插值
    4. 原始分数高则加成，低则减成
    """
    if total <= 1:
        return 90
    
    # 位置基础分（排名越好，分数越高）
    position_base = 95 - (rank * 20 / total)
    
    # 原始分数调整（考虑相对质量）
    if raw_score >= 0.5:
        adjustment = 5  # 高分加成
    elif raw_score >= 0.35:
        adjustment = 0  # 中等不变
    else:
        adjustment = -5  # 低分减成
    
    display = position_base + adjustment
    
    # 确保在合理范围
    return int(max(65, min(98, display)))
```

### 5.3 示例映射

| rank | total | raw_score | position_base | adjustment | display_score |
|------|-------|-----------|---------------|------------|---------------|
| 0 | 5 | 0.52 | 95 | +5 | 95 |
| 1 | 5 | 0.38 | 91 | 0 | 91 |
| 2 | 5 | 0.30 | 87 | -5 | 82 |
| 3 | 5 | 0.25 | 83 | -5 | 78 |
| 4 | 5 | 0.21 | 79 | -5 | 74 |

## 六、结论

原始排序分数（similarity / hybrid_score / cf_score）是技术指标，不适合直接面向用户展示。需要通过校准函数转换为用户友好的 display_score，满足：

1. Top 结果显示 85~95
2. 中间结果显示 75~88
3. 较弱结果显示 65~80
4. 保持批内层次感
5. 不影响原始排序逻辑
