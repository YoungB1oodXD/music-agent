# 最保险演示流程

> 适用于中期检查、答辩现场、功能演示等需要高成功率的场景
> 预计演示时间：2-3 分钟

---

## 核心原则

**不要演"跨流派硬切换"，演"场景 + 能量 + 声乐"的多轮修正。**

原因：
- 流派切换在当前实现里是累积偏好，不是强覆盖
- "不要摇滚要电子"这类话术容易残留旧流派
- 场景/能量/声乐的识别更稳定，测试覆盖更全

---

## 最保险演示流程（5 步）

| 步骤 | 操作 | 输入内容 | 预期效果 |
|:---:|:---:|:---|:---|
| 1 | 输入 | 推荐一些适合学习的纯音乐 | 场景=学习，声乐=纯音乐，返回 5 首 |
| 2 | 输入 | 我想要更欢快一点，最好适合跑步 | 场景=跑步，能量=high，返回新推荐 |
| 3 | 输入 | 不要人声，来点纯音乐 | 声乐=纯音乐，保持跑步+高能量 |
| 4 | 点击 | 换一批按钮 | 当前偏好不变，返回不重复的新结果 |
| 5 | 输入 | 为什么推荐这些歌？ | 解释文本，不替换当前推荐列表 |

---

## 详细演示步骤

### 第 1 步：建立稳定初始场景

**输入：**
```
推荐一些适合学习的纯音乐
```

**预期：**
- 右侧出现 5 首推荐
- 系统识别场景为"学习"
- 声乐偏好为"纯音乐"

**台词：**
> 我先给系统一个明确的学习场景和纯音乐偏好，建立第一轮推荐上下文。

---

### 第 2 步：场景 + 能量修正

**输入：**
```
我想要更欢快一点，最好适合跑步
```

**预期：**
- 直接触发新推荐（不需要再点换一批）
- 场景从"学习"切换到"跑步"
- 能量偏好更新为"high"
- 推荐列表更新

**台词：**
> 这一步是在上一轮基础上继续修正需求。系统会把场景从学习切换到跑步，并提高能量偏好。这是多轮对话主导的体现。

---

### 第 3 步：声乐偏好细化

**输入：**
```
不要人声，来点纯音乐
```

**预期：**
- 再次触发新推荐
- 保持"跑步 + 高能量"不变
- 叠加"纯音乐"偏好

**台词：**
> 继续追加限制条件，不是推翻之前的需求，而是在当前偏好基础上进一步收窄结果。

---

### 第 4 步：换一批（翻页）

**操作：**
点击界面上的"换一批"按钮

**预期：**
- 当前偏好不变（跑步、高能量、纯音乐）
- 返回新的 5 首歌
- 与上一批不重复

**台词：**
> 这里的"换一批"不是重新理解需求，而是在当前已确定的偏好下继续翻页，展示另一组结果。

---

### 第 5 步：解释能力展示

**输入：**
```
为什么推荐这些歌？
```

**预期：**
- 系统返回解释文本
- 当前推荐列表保持不变（不被替换）

**台词：**
> 系统除了能多轮修正推荐，还能解释当前为什么推荐这一批结果。

---

## 完整台词版（可直接照念）

> 我现在演示的是一个多轮对话音乐推荐系统。它不是每轮重新开始，而是会持续记住当前会话里的场景、能量和声乐偏好。
>
> 我先输入：**推荐一些适合学习的纯音乐**。
>
> 现在我继续细化需求：**我想要更欢快一点，最好适合跑步**。这一步不是新开一轮，而是在已有偏好上做修正。
>
> 接着我再加一个限制：**不要人声，来点纯音乐**。系统会在"跑步、高能量"的基础上，再进一步收敛到器乐方向。
>
> 现在我点击 **换一批**。这里展示的是在当前偏好不变的情况下，系统换一组新的候选结果。
>
> 最后我问一句：**为什么推荐这些歌？** 这一步展示系统除了推荐，还能对当前结果做解释。

## 风险提示

### 不建议在主演示中使用的话术

| 风险话术 | 风险原因 |
|:---|:---|
| 不要摇滚，要电子音乐 | 流派是累积偏好，容易残留旧流派 |
| 我想听爵士 | 同上，流派切换不够稳定 |
| 类似第一首那种风格的 | 依赖 liked_songs 元数据，不够保险 |
| 我不喜欢这首歌 | 可能默认落到列表第一首，而非当前播放的歌 |
| 再来点类似的 | 意图不够明确，可能复用旧 query |

### 如果想展示反馈

**推荐做法：**
- 直接点击界面上的"喜欢"或"不喜欢"按钮
- 不要用自然语言说"我喜欢这首"

**原因：**
按钮触发更稳定，自然语言反馈可能有歧义。

## 为什么这套流程最保险

1. **每一步都有明确的 slot 触发词**
   - 学习、跑步 → scene
   - 欢快、跑步、运动 → energy=high
   - 不要人声、纯音乐 → vocals=instrumental
   - 换一批 → refresh
   - 为什么 → explain

2. **已被测试覆盖**
   - 这套流程在 `tests/test_multi_turn_refinement.py` 中有完整覆盖
   - 每一步都有断言验证

3. **符合当前实现的多轮对话语义**
   - 偏好修正确实会直接触发新推荐
   - 不需要额外点"换一批"才能生效
   - 换一批确实只是翻页，不改偏好
