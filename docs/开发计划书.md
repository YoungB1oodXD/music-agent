# Music Agent 智能音乐推荐系统开发计划书

## 1. 项目概述
本项目旨在构建一个基于「双脑架构」的智能音乐推荐对话助手。通过结合语义搜索（BGE-M3 + ChromaDB）与协同过滤（Implicit ALS），并集成大语言模型（LLM）作为中枢调度，实现理解自然语言意图、多轮交互反馈以及可解释性推荐的完整闭环。

## 2. 核心目标
- **意图识别与槽位提取**：准确识别用户音乐需求（风格、心情、场景等）。
- **工具化调度**：通过 Function Calling 机制调度语义搜索与协同过滤引擎。
- **RAG 增强生成**：基于检索到的音乐元数据，生成带有引用的推荐理由。
- **基础多轮状态管理**：维护会话上下文，支持基础的意图继承与槽位更新（增强型多轮管理见 Milestone 2）。

## 3. 开发里程碑 (Waves)

### Wave 1: 基础设施与协议 (Foundation) - DONE
- **LLM 客户端实现**：对接通义千问（Qwen）OpenAI 兼容接口，支持流式与非流式切换。
- **协议定义**：确定意图识别、槽位提取及最终响应的 JSON Schema。
- **工具注册表**：建立统一的工具调度框架，封装现有的搜索与推荐 API。

### Wave 2: 智能体核心 (Agent Loop) - DONE
- **编排器实现**：开发 Orchestrator 状态机，管理从意图识别到工具调用再到响应生成的全流程。
- **RAG 增强**：实现上下文构建器，对检索结果进行安全过滤（防注入）与长度截断。
- **确定性模拟**：提供 Mock LLM 模式，确保在无 API Key 环境下核心逻辑可验证。

### Wave 3: 交互界面与演示 (Surfaces) - DONE
- **命令行交互 (CLI)**：提供支持多轮对话的 `chat_cli.py`。
- **回放与审计**：实现对话记录（Transcript）持久化与重放功能，便于调试。
- **演示脚本**：恢复并增强 `demo.py` 与 `demo_safe.py`，对齐开题答辩要求。

### Milestone 2: 增强型多轮对话状态管理 (Planned)
- **对话历史感知**：将完整对话历史作为推理上下文，支持“换一批”等模糊指令。
- **细粒度偏好微调**：支持“不要太吵”、“来点纯音乐”等负向反馈与风格过滤。
- **鲁棒性增强**：优化意图切换与槽位冲突处理逻辑。

## 5. 关键风险与应对

| 风险描述 | 影响程度 | 应对措施 |
| :--- | :--- | :--- |
| **Windows 兼容性** | 高 | `implicit` 库在 Windows 下易卡死，默认在演示脚本中禁用 CF 或使用线程隔离。 |
| **API 密钥缺失** | 中 | 提供完整的 Mock 模式，使用预定义的确定性响应完成流程演示。 |
| **模型产物缺失** | 高 | 在初始化阶段进行严格路径检查，若 `index/` 或 `data/models/` 不存在则给出清晰指引。 |
| **Prompt 注入** | 低 | 在 RAG 上下文构建阶段实施关键词过滤，移除潜在的指令劫持文本。 |

## 6. 运行与演示指南

### 6.1 环境准备
确保已安装 `requirements.txt` 中的依赖，并配置环境变量（可选）：
```bash
# 配置通义千问 API Key (若需真实调用)
set DASHSCOPE_API_KEY=your_api_key_here
```

### 6.2 核心功能演示
使用 Mock 模式运行单次推荐测试：
```bash
python scripts/chat_cli.py --llm mock --once "推荐点适合学习的歌"
```

查看最近一次对话的回放：
```bash
python scripts/replay_transcript.py --latest
```

### 6.3 答辩演示脚本
针对不同环境提供两个演示入口：
- **安全版**（离线/无 Key）：`python demo_safe.py`
- **完整版**（在线/有 Key）：`python demo.py`

## 7. 任务书要求对齐
本项目开发计划严格遵循任务书中的四项核心要求：
1. **语义搜索**：通过 `MusicSearcher` 实现基于向量空间的自然语言检索。
2. **协同过滤**：通过 `MusicRecommender` 实现基于用户行为模式的关联推荐。
3. **多轮交互**：通过 `SessionState` 与 `Orchestrator` 实现基础对话上下文维护（增强型多轮管理规划中）。
4. **大模型集成**：利用 LLM 作为中枢，实现意图理解与推荐理由的自然语言生成。

## 8. 验证与质量保证
在提交代码前，必须通过以下自动化检查：
- **编译检查**：`python -m compileall src scripts tests -q`
- **冒烟测试**：运行 `tests/simulate_session.py` 验证基础流程。
- **Schema 验证**：确保所有 LLM 输出符合预定义的 Pydantic 模型。
